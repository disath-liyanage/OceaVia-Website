Home Page

Technical Description

The OceaVia Home Page is structured using semantic HTML5 elements such as <header>, <section>, <main>, and <footer> for clear layout and improved accessibility. A full-screen background video is implemented using the <video> element with autoplay, loop, and muted attributes to ensure a smooth, silent visual introduction without interrupting users. Overlaying text and buttons (.hero-overlay-content) are positioned using Flexbox and enhanced with background gradients, -webkit-background-clip, and text-fill-color for aesthetic appeal.

The mission section uses an <h1> heading followed by multiple <p> tags, styled with consistent font sizing, padding, and line height for readability. The “Explore More” section uses a CSS grid layout via display: grid and grid-template-columns to create a responsive card layout. Each card is built with a combination of <img>, <div>, and <a> elements, styled to include hover effects such as transform: scale(), filter: brightness(), and shadow effects for interactivity.

The navigation bar uses Flexbox for horizontal alignment and is fixed using position: fixed to keep it always visible. Reusable utility classes and global CSS are applied for consistent spacing and typography across the page.

Key HTML tags used:

  • <header>, <nav>, <section>, <main>, <footer> – for semantic page structure
  • <video>, <h1>, <p>, <img>, <a> – for media display and content interaction
  • <div> – for grouping and styling layout elements

Key CSS properties used:

  • display: flex, justify-content, align-items – for alignment in navbar and hero section
  • display: grid, grid-template-columns – for responsive card layout
  • position: absolute, fixed, relative, z-index – for layering and fixed elements
  • object-fit: cover – to scale video/image without distortion
  • text-shadow, box-shadow, transform, transition – to enhance hover effects and animations
  • @media queries – to ensure mobile responsiveness

Accessibility

Accessibility is addressed through descriptive alt attributes on all <img> elements, including logos and cards. The video uses muted, playsinline, and fallback text to accommodate assistive technologies. Color contrast is optimized using light text on dark or video backgrounds with shadow support for legibility. The use of semantic tags like <header>, <section>, and <footer> improves document structure for screen readers. All interactive elements are accessible using keyboard navigation.

Table Page

Technical Description

The OceaVia Table Page presents a clean, responsive, and data-focused interface showcasing information about threatened marine species. It uses semantic HTML5 elements including <main>, <table>, and <thead>/<tbody> to structure tabular content. A custom-designed table layout visually organizes species data by risk level, region, and related action links.

The table header uses rowspan and colspan attributes to align content neatly across multiple columns. Each row uses a unique class such as .critical, .endangered, .vulnerable, and .near to dynamically apply background colors and badges based on risk level. Hover effects are added to table rows using transform and box-shadow for interactive depth.

Table responsiveness is enhanced by limiting the width to 95% and centering it using margin auto. Semantic headings and spacing ensure clean readability. The navigation bar and footer follow the same global style as the rest of the site, ensuring visual and structural consistency.

Key HTML tags used:

  • <main>, <table>, <thead>, <tbody>, <tr>, <td> – for structured tabular content
  • <img>, <a> – for species icons and external links
  • <header>, <footer> – for consistent layout

Key CSS properties used:

  • border-collapse, padding, text-align – to enhance table presentation
  • box-shadow, transform, transition – for row hover animations
  • background-color – used with specific row classes to indicate risk level
  • display: block, margin: auto – for centering logos and icons

Accessibility

The table page includes descriptive alt text for all image elements, including species logos and icons, to support screen readers. Color-coded rows are paired with bold text and uppercase badges to improve visual clarity and contrast. All external links open in a new tab using target="_blank", and semantic HTML ensures assistive technologies can interpret the document structure accurately.

Content Page – Marine Plastic Pollution

Technical Description

The Content Page on Marine Plastic Pollution is designed as a comprehensive, visually engaging, and accessible layout. It uses semantic HTML5 structure elements including <main>, <section>, <figure>, and <table> to present researched data in a structured and readable format. The layout includes a fixed-position sidebar for internal navigation and a scroll-activated section reveal animation, enhancing both usability and engagement.

The sidebar navigation uses a nav element with anchor links, highlighting the current section dynamically with JavaScript based on scroll position. Each content section is tagged with an id to enable smooth anchor-based navigation and is revealed with a fade-in animation triggered by scroll events using classList.add('visible').

The page follows a content-rich design, with scroll-behavior smoothness and hover interactions on figures and table rows. Data is represented using a responsive <table> for environmental impacts and a .stats-grid CSS Grid layout for key statistics. This approach offers a balance between text and visual content, optimized for comprehension and interaction.

Key HTML tags used:

  • <main>, <section>, <table>, <figure> – to organize content and data visually
  • <img>, <figcaption>, <ul>, <li> – to enhance accessibility and structure
  • <nav> (with aria-label) – for accessible navigation
  • <footer> – to maintain consistent branding and link back to editor and validation pages

Key CSS properties used:

  • display: flex, grid-template-columns, max-width – for responsive page and content layout
  • background, box-shadow, border-radius – for polished, modern visual aesthetics
  • scroll-behavior: smooth, transform, transition – for scroll animations and hover effects
  • @media queries – for mobile responsiveness (e.g., sidebar layout adapts on smaller screens)

Accessibility

The page adheres to accessibility best practices by providing meaningful alt attributes for all images, using semantic landmarks such as <main> and <nav>, and maintaining high contrast in text and links. Sidebar navigation includes an aria-label for screen readers. Section headings are properly ordered to allow assistive technologies to parse the document effectively. Link hover states are visually enhanced for clarity.

Challenges and Lessons Learned

One of the key challenges during this project was ensuring that the content layout remained visually consistent across different screen sizes. Initially, some sections appeared misaligned or overlapped on smaller devices, which required revisiting our CSS structure and applying media queries more effectively.

Another issue we faced was achieving smooth internal navigation with a fixed sidebar and header. The anchor links didn’t scroll to the correct positions due to fixed element heights. This was resolved by using scroll-padding-top in CSS and testing across various browsers.

Through these challenges, we learned the importance of planning for responsiveness from the beginning and thoroughly testing navigation and layout features. We also gained a deeper understanding of combining semantic HTML with modern CSS techniques to enhance both usability and accessibility.

Compliance

All pages created for this website follow the Janet acceptable use policies and guidelines for academic web publishing. Content is appropriate for an educational setting, with no offensive, misleading, or commercial material.

Images used are either sourced from royalty-free platforms or designed specifically for the project. Each image includes descriptive alt text to support accessibility for screen readers, meeting WCAG standards. External links open in a new tab using target="_blank" and are clearly labeled to avoid confusion.

The website uses semantic HTML5 and CSS to improve accessibility, navigation, and compatibility across devices. No copyrighted or unlicensed media is included, and all content is student-generated or properly referenced. This ensures that the site remains respectful of intellectual property and suitable for hosting on an academic network.
